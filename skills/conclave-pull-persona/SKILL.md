# Conclave Pull Persona

ユーザースコープのペルソナをプロジェクトへマージする。  
git pull のイメージ: ユーザースコープ（リモート）→ プロジェクト（ローカル）。

## 入力

以下のいずれかを受け付ける:

- `--persona <id>`: 単一ペルソナを対象
- `--guild <guild>`: ギルド内の全ペルソナを対象
- 引数なし: ユーザースコープの全ペルソナを候補として表示し、人間が選択

## ワークフロー

1. `.conclave/config.yaml` を読む。以下を決定する:
   - `sync.user_scope`（既定: `~/.conclave`）
   - `sync.conflict_strategy`（既定: `stop`）

2. `<user_scope>/personas/` を走査し候補ペルソナを探す。
   `--persona` または `--guild` が指定されていれば、それに応じて絞り込む。

3. 各候補を分類する:
   - **新規追加候補**: `id` が `.conclave/personas/` に存在しない
   - **差分マージ候補**: `id` が既に `.conclave/personas/` に存在する

4. 分類済みリストをユーザーに提示し、進める前に確認を求める:
   ```
   ## Pull Persona — 候補一覧

   ### 新規追加候補 (<N>件)
   - <guild>/<id>: <role.ja>

   ### 差分マージ候補 (<N>件) — growth.yaml のみ
   - <guild>/<id>: <role.ja>

   続行しますか? [y/N]
   ```

5. 各**新規追加候補**について（ユーザーが確認した場合）:
   a. ユーザースコープの `identity.yaml` に対して L1 必須フィールド検証を
      実行する。
   b. 既存のプロジェクトペルソナに対して L2 類似度チェックを実行する。
      閾値を超える場合はユーザーに警告し、続行するか確認する。
   c. プロジェクトのペルソナ容量（`limits.max_personas`,
      `limits.max_personas_per_guild`）を確認する。容量超過の場合はブロックする。
   d. `identity.yaml` を `.conclave/personas/<guild>/<id>/identity.yaml` に
      コピーする。
   e. `growth.yaml`（存在する場合）をそのままコピーする。
   f. **追加**としてマークする。

6. 各**差分マージ候補**について（ユーザーが確認した場合）:
   a. **identity.yaml**: 自動的に上書きしない。
      ユーザースコープの identity とプロジェクトの identity を比較する。
      差異があれば diff を表示し、尋ねる:
      「identity.yaml に差異があります。プロジェクト版を上書きしますか? [y/N]」
      ユーザーが明示的に承認した場合のみ書き込む。
   b. **growth.yaml**: エントリ単位でマージする:
      - エントリ `id` がプロジェクトの growth に無い → 追記（**追加**とマーク）。
      - `id` が存在し内容が同一 → スキップ（**スキップ**とマーク）。
      - `id` が存在し内容が異なる → `conflict_strategy` を適用する:
        - `stop`: 停止し、競合を報告し、人間を待つ。
        - `skip`: プロジェクト版を維持し、**スキップ**とマークする。
        - `ask`: diff を表示し、ユーザーにユーザースコープ版 / プロジェクト版 /
          手動 を選ばせる。

7. 完了レポートを表示する:
   ```
   ## Pull Persona レポート

   対象: <N> ペルソナ

   | ペルソナ | 種別 | identity | growth 追加 | growth スキップ | 要確認 |
   |---|---|---|---|---|---|
   | <id> | 新規 / マージ | 追加 / 更新 / スキップ | N | N | N |

   完了。プロジェクトスコープ: .conclave/personas/
   ```

## ルール

- 明示的な人間承認なしにプロジェクトの `identity.yaml` を絶対に上書きしない。
- 既存のプロジェクトペルソナを自動的に削除しない。
- 全ての新規ペルソナに対して L1・L2 チェックは必須。
- 既存の growth エントリを全て保持する。追記のみ行い、削除しない。
- L1 検証に失敗した場合、そのペルソナはスキップし報告する。
- Conclave は判断材料を提供するだけであり、ユーザーに代わって決定はしない。
