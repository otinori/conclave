# 最初のユースケース: 設計判断レビュー

最初は範囲を絞った設計判断のレビューに Conclave を使う。

## 標準の召喚メンバー

- `yamamoto-takuya`: アーキテクチャと契約構造
- `sato-kenichi`: 運用とアセット管理

以下は論点がそのレンズを必要とするときだけ追加する:

- `watanabe-mariko`: 機密性・知財・ガバナンス
- `tanaka-misaki`: 現場での使いやすさ
- `nakamura-takashi`: 知識戦略と組織適合
- `hayashi-chinatsu`: エンドユーザーの分かりやすさと採用リスク
- `tadokoro-takuma`: プロダクトオーナーシップと収益判断

## セッションの型

1. 議題と機密区分を宣言する。
2. 2名のペルソナを召喚する。
3. それぞれの初期見解を求める。
4. 対立点を1つ挙げてもらい、あり得る調停案を1つ求める。
5. セッションを終了し、勤怠・議事録を生成する。
6. 十分なセッションが安定した記憶の種を生んでから蒸留する。

目安時間: 15〜30分。
